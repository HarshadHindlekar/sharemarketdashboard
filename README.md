# Sharemarket Dashboard

Monorepo for a share-market dashboard with:

- `client/`: Angular standalone-components app with routing and Tailwind CSS.
- `server/`: Node.js, Express, and TypeScript API server.

## Folder Structure

```text
sharemarket-dashboard/
|-- client/        # Angular app generated by Angular CLI
|-- server/        # Express API written in TypeScript
|-- package.json   # Root npm workspace and shared scripts
|-- .editorconfig  # Cross-editor formatting defaults
|-- .gitignore     # Files Git should ignore
`-- README.md      # Project setup notes
```

## Exact Scaffold Commands

Angular CLI `22.0.5` is the current latest stable CLI resolved during setup. It requires Node `^22.22.3`, `^24.15.0`, or `>=26.0.0`.

```powershell
mkdir sharemarket-dashboard
cd sharemarket-dashboard
git init

npm init -y

npx -y @angular/cli@latest new client --directory client --standalone --routing --style css --skip-git --package-manager npm --defaults
cd client
npx -y @angular/cli@latest ng add tailwindcss --skip-confirmation
cd ..

mkdir server
cd server
npm init -y
npm install express cors dotenv
npm install -D typescript tsx @types/node @types/express @types/cors
cd ..

npm install -D concurrently
```

If your installed Node is slightly below Angular's required range, this no-global-change variant runs the Angular CLI with a temporary compatible Node:

```powershell
npx -y -p node@24.15.0 -p @angular/cli@latest ng new client --directory client --standalone --routing --style css --skip-git --package-manager npm --defaults
cd client
npx -y -p node@24.15.0 -p @angular/cli@latest ng add tailwindcss --skip-confirmation
cd ..
```

## Workspace Scripts

Run both apps in development:

```powershell
npm run dev
```

Run only the client:

```powershell
npm run dev:client
```

Run only the server:

```powershell
npm run dev:server
```

Build both workspaces:

```powershell
npm run build
```

## Config Files

- `package.json`: The root npm workspace manifest. It marks the repo private, registers `client` and `server` as workspaces, and provides scripts for running, building, and testing packages from one place.
- `client/angular.json`: Angular CLI workspace configuration. It defines how the client app is built, served, tested, and where source/build assets live.
- `client/package.json`: Client dependencies and Angular scripts. `ng serve` runs the app, `ng build` creates production output, and `ng test` runs client tests.
- `client/tsconfig.json`, `client/tsconfig.app.json`, `client/tsconfig.spec.json`: TypeScript settings for the Angular workspace, browser app build, and test build.
- `client/.postcssrc.json`: PostCSS configuration generated by `ng add tailwindcss`; it loads Tailwind's PostCSS plugin for Angular builds.
- `client/src/styles.css`: Global Angular stylesheet. It imports Tailwind with `@import 'tailwindcss';`.
- `server/package.json`: Server dependencies and scripts. `dev` runs TypeScript directly with `tsx watch`, `build` emits JavaScript to `dist`, and `start` runs the compiled server.
- `server/tsconfig.json`: TypeScript compiler settings for the Express server. It uses Node's modern ESM resolution and outputs compiled files to `server/dist`.
- `.gitignore`: Keeps dependencies, build output, logs, local environment files, and editor/OS noise out of Git.
- `.editorconfig`: Shared formatting defaults for editors: UTF-8, LF line endings, two-space indentation, final newline, and trimmed trailing whitespace.
